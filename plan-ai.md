# План: вёрстка «Recipe page» (Frontend Mentor)

## Контекст

Это учебный челлендж Frontend Mentor — страница рецепта омлета. Стартовый
`index.html` содержит только сырой текст без разметки. Цель — сверстать страницу
по макету (Mobile / Tablet / Desktop) максимально точно, используя точные
значения из Figma.

**Формат работы (по твоему запросу и по `AGENTS.md`):**
- Код пишешь **ты**. Я **не пишу за тебя готовый код** — даю теорию, объясняю на
  примерах, показываю лучшие практики и проверяю результат.
- Язык общения — **русский** (термины дублирую по-английски).
- Точность — берём **реальные значения из Figma** (см. справочник токенов ниже).
- Используем **CSS-переменные** (`:root` + `var()`) с самого начала.
- На каждом этапе даю 4 вещи: **теория → пример реализации → лучшие практики →
  почему именно так**. После каждого этапа проверяю код на ошибки/опечатки,
  соответствие макету и пропущенные элементы.

## Справочник дизайн-токенов (из Figma + style-guide)

### Цвета
| Переменная | Значение | Где используется |
|---|---|---|
| `--white` | `hsl(0 0% 100%)` | фон карточки |
| `--stone-100` | `hsl(30 54% 90%)` | фон страницы |
| `--stone-150` | `hsl(30 18% 87%)` | разделительные линии |
| `--stone-600` | `hsl(30 10% 34%)` | основной текст |
| `--stone-900` | `hsl(24 5% 18%)` | заголовок h1 |

| `--rose-800` | `hsl(332 51% 32%)` | заголовок «Preparation time» |
| `--rose-50` | `hsl(330 100% 98%)` | фон блока «Preparation time» |

### Шкала отступов (одна шкала на всё — gap/padding)
`8px` · `12px` · `16px` · `24px` · `32px` · `40px`
(в Figma это токены spacing 100/150/200/300/400/500 — заведём как переменные).

### Типографика
| Назначение | Шрифт | Размер | Начертание | line-height |
|---|---|---|---|---|
| h1 (заголовок рецепта) | Young Serif | 36px (mobile) / 40px (tablet+desktop) | 400 | 1 |
| h2 (Ingredients/Instructions/Nutrition) | Young Serif | 28px | 400 | 1 |
| «Preparation time» | Outfit | 20px | 600 | 1 |
| Основной текст | Outfit | 16px | 400 | 1.5 |
| Жирные подписи (Total/Beat the eggs и т.п.) | Outfit | 16px | 700 | 1.5 |

Шрифты лежат локально в `assets/fonts/` (Young Serif 400; Outfit 400/600/700).

### Раскладка по брейкпоинтам
| | Mobile (375) | Tablet (768) | Desktop (1440) |
|---|---|---|---|
| Карточка | во всю ширину, без скруглений | по центру, `max-width ≈ 616px` | по центру, `max-width 736px` |
| Внешний отступ страницы | 0 | сверху/снизу ~128px | сверху/снизу ~128px |
| Внутр. падинг карточки | 0 (только текст имеет 32px по бокам) | 40px | 40px |
| Картинка | во всю ширину, углы прямые, без скруглений | 12px скругление | 12px скругление |

### Ключевые расстояния внутри карточки
- Карточка: `padding 40px`, между картинкой и текстом `gap 40px`, скругление `24px`.
- Текстовый блок: между секциями `gap 32px`.
- title: между h1 и абзацем `gap 24px`.
- Блок preparation: `padding 24px`, скругление `12px`, между заголовком и списком `gap 16px`.
- Списки (ingredients/instructions/preparation): между пунктами `gap 8px`;
  отступ маркера слева `8px`, расстояние маркер→текст `16px`.
- Заголовок секции → её содержимое: `gap 24px`.
- Таблица nutrition: между строками `gap 12px`; горизонтальный паддинг строки `32px`;
  тонкие линии между строками (`--stone-150`), под последней строкой линии нет.

## Этапы (по ним движемся последовательно)

> На каждом этапе сначала разбираем теорию и план, потом **ты пишешь код**, потом я
> проверяю. Переходим к следующему, только когда текущий блок готов.

### Этап 0 — Подготовка проекта
- Завести отдельный `style.css` и подключить его в `<head>` (`<link rel="stylesheet">`).
- Очистить плейсхолдер-текст из `<body>` (он заменится разметкой на Этапе 1).
- Базовый сброс: `box-sizing: border-box`, обнуление `margin` у `body`.
- **Теория:** разделение содержимого (HTML) и оформления (CSS), зачем нужен reset и `box-sizing`.

### Этап 1 — Семантическая HTML-разметка (контент без стилей)
- `main` → `article` (карточка) → `figure/img`, `h1`, абзац-интро.
- Секции: preparation, ingredients, instructions, nutrition (`section` + `h2`).
- Списки: `ul` для ingredients/preparation, `ol` для instructions, `table` для nutrition.
- Жирные подписи — `<strong>`; осмысленный `alt` у картинки.
- **Проверка:** иерархия заголовков (один h1), корректные семантические теги, alt-текст.

### Этап 2 — Базовые стили: переменные, шрифты, body
- `:root` с переменными цветов, отступов и шрифтов (из справочника выше).
- Подключение шрифтов (`@font-face` локальных файлов — рекомендуется; альтернатива — Google Fonts).
- `body`: фон `--stone-100`, шрифт Outfit 16px/1.5, цвет `--stone-600`, центрирование карточки.
- **Теория:** CSS-переменные и зачем они, наследование шрифтовых свойств, относительные единицы (rem).

### Этап 3 — Карточка-контейнер
- `--white` фон, скругление 24px, padding 40px, `max-width 736px`, центрирование (`margin-inline: auto`).
- Внутренняя раскладка — `display: flex; flex-direction: column; gap: 40px`.
- **Теория:** блочная модель («коробка с подарком»), Flexbox-контейнер и `gap`.

### Этап 4 — Изображение
- Адаптивная картинка: `display: block; width: 100%; height: auto` (или `object-fit: cover`), скругление 12px.
- **Теория:** почему `display:block` и `width:100%`, чем `object-fit` отличается.

### Этап 5 — Заголовок + интро
- h1 Young Serif (36/40px), цвет `--stone-900`; абзац `--stone-600`; `gap 24px`.

### Этап 6 — Preparation time (розовый блок)
- Фон `--rose-50`, скругление 12px, padding 24px; заголовок `--rose-800` 20px/600.
- Список с кастомными маркерами-точками, расстояния 8px/16px.
- **Теория:** оформление маркеров списка (`::marker` / кастомные буллеты), внутренние отступы блока.

### Этап 7 — Ingredients
- h2 `--brown-800` 28px; маркированный список, цвет маркеров `--brown-800`, gap 8px.

### Этап 8 — Instructions
- h2; нумерованный список `ol`; жирные подписи `strong`; номера `--brown-800`.
- **Теория:** разница `ul`/`ol`, стилизация номеров.

### Этап 9 — Nutrition + таблица
- h2 + абзац-пояснение; таблица: 2 колонки 50/50, паддинг строк, нижние границы строк
  (`--stone-150`), без линии под последней; значения `--brown-800` 700.
- **Теория:** семантика `table`, границы и `border-collapse`, `:last-child`.

### Этап 10 — Разделительные линии
- Тонкие линии (`--stone-150`) между ingredients/instructions/nutrition (`<hr>` или `border`).

### Этап 11 — Адаптивность (mobile → tablet → desktop)
- Подход mobile-first: базовые стили = mobile (карточка во всю ширину, картинка без скруглений).
- Медиа-запрос (~`640px`+): карточка получает `max-width`, отступы по краям, скругления, картинка скругляется.
- **Теория:** mobile-first, медиа-запросы, относительные единицы для адаптивности.

### Этап 12 — Финальная проверка и доступность
- Сверка с макетами всех трёх размеров; контраст цветов; видимый фокус для клавиатуры.
- Валидация HTML, заполнение блока `attribution` (имя/ссылка).

## Дополнительные свойства для тренировки

По твоему запросу на каждом блоке проговариваем и осознанно прописываем «общие»
свойства, даже если значение по умолчанию подходит: `display`, `box-sizing`,
`margin`/`padding`, `width`/`max-width`, `gap`, `color`, `font-*`, `line-height`,
`border`, `border-radius`, `background`, `list-style`. Цель — закрепить понимание,
а не просто получить результат.

## Как проверяем результат (verification)

1. Открыть `index.html` в браузере (через Live Server или просто файлом).
2. Включить DevTools → device toolbar и проверить ширины **375px, 768px, 1440px**
   и промежуточные (от 320px и шире).
3. Сравнить с `design/mobile-design.jpg` и `design/desktop-design.jpg`; для точных
   значений сверяться с Figma.
4. Проверить доступность: контраст, навигацию с клавиатуры (видимый фокус),
   корректность alt-текста и иерархии заголовков.
5. Прогнать HTML через валидатор (validator.w3.org) — ноль ошибок.
